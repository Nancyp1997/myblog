NoSQL databases comparison

Document Graph Key-value Time series Wide column
Why? To store data in document oriented model in independent documents. Complex relations in relational DB make responses slow. To speed up the response times, data that needs complex joins for meaningful results are stored in graph databases, where tables are vertices and edges are the relationships like in RDBMS. Fetch in O(1) time. Fetch data with minimum latency. Used to track & persist data that is continually read & written over time period. Handle big data. Perfect for analytics use cases. Highly performant and scalable.
When to pick this? Pick this when you need a flexible schema that changes often & when you need to scale fast alongside staying highly available. Pick this for network graphs, recommendation engines, fraud analysis, genetic data use cases. Whenever you need to store complex relationships, consider graph database. Pick this to fetch data fast with minimum latency. To store time series data. Pick this to handle big data analytics. Ensures scalability, performance and high availability.
Use cases Real-time feeds, live-sports apps, inventory management, storing user comments, web-based multiplayer games. Relationships between users in social networks, maps. Used in caching, persisting user state & user sessions, managing real time data & implementing queues. Creating leaderboards in online games and web apps. Implementing a pub-sub system. Handling real time data like data from iOT devices. Managing data for running monitoring & analytics. Writing an autonomous trading platform to see trends in stock prices in real time.
Enterprise products MongoDB, CouchDB, OrientDB, Google cloud datastore & Amazon documentDB. Neo4J Redis, Hazelcast, Riak, Voldemort and Memcached. InfluxDb, Timescale DB, Prometheus Cassandra, HBase, Google BigTable, ScyllaDB
Companies SEGA uses MongoDB. Coinbase scaled from 15k to 1.2 mil requests per minute with MongoDB. Walmart shows product recommendations to its customers using Neo4J graph DB. NASA uses Neo4J to store lessons learned data from previous missions. Microsoft uses Redis to handle traffic spikes on its platforms. Google cloud uses Memcached to implement caching on their cloud platform. IBM uses InfluxDB for real time fraud detection analytics. Spiio uses InfluxDB to monitor plant installations. Netflix uses Cassandra in analytics infrastructure. Adobe uses HBase for processing large amounts of data.
